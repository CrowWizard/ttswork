# 语音工作台 UI 重构与声纹独立计划

## 需求概览

1. 用户信息提取为顶部固定导航栏 Header
2. 音频上传和开始录音互换位置，提示上传声音效果更好
3. 声纹从用户表独立为 VoiceProfile 表
4. 录音/上传后自动生成声纹（后端异步 + 前端轮询）
5. 录音上传只显示最后一条
6. 修改 Step 和其他文案

---

## 一、数据库 Schema 变更

### 新增 `VoiceProfile` 模型（两个 schema 文件同步修改）

**文件**: `prisma/schema.prisma` + `api-server/prisma/schema.prisma`

```prisma
model VoiceProfile {
  id                            String   @id @default(cuid())
  userId                        String?  @unique
  anonymousUserId               String?  @unique
  activePureVoiceEnrollmentId   String?
  activeSceneVoiceEnrollmentId  String?
  createdAt                     DateTime @default(now())
  updatedAt                     DateTime @updatedAt

  @@index([userId])
  @@index([anonymousUserId])
}
```

### 修改 `User` 模型

移除字段：
- `activePureVoiceEnrollmentId`
- `activeSceneVoiceEnrollmentId`

### 修改 `AnonymousUser` 模型

移除字段：
- `activePureVoiceEnrollmentId`
- `activeSceneVoiceEnrollmentId`

### 迁移步骤

1. 创建 `VoiceProfile` 表
2. 从 `User`/`AnonymousUser` 迁移现有 `activePureVoiceEnrollmentId`/`activeSceneVoiceEnrollmentId` 数据到 `VoiceProfile`
3. 移除 `User`/`AnonymousUser` 中的对应字段
4. 执行 `npx prisma db push` 或 `npx prisma migrate dev`

## 二、后端 API 变更

### 2.1 声纹路由改为异步处理（需求 4）

**文件**: `api-server/src/routes/voice-enrollments.ts`

**当前流程**: POST 创建 enrollment → 同步调用 Qwen → 更新状态为 READY → 返回

**新流程**:
1. POST `/api/voice/enrollments` 创建 enrollment（状态 PENDING）→ 立即返回
2. 启动后台异步任务调用 Qwen 建声
3. 成功后更新状态为 READY，失败更新为 FAILED
4. 前端轮询 GET `/api/voice/profile` 获取最新状态

具体修改：
- `voice-enrollments.ts` POST handler: 创建 enrollment 后立即返回，不等待 Qwen
- 使用 `void` 启动异步处理函数（`processEnrollmentAsync`）
- 异步函数内部：调用 Qwen → 更新 enrollment 状态 → 更新 VoiceProfile

### 2.2 录音上传后自动触发双声纹生成（需求 4）

**文件**: `api-server/src/routes/voice-recordings.ts`

在录音上传成功后，自动创建两条 VoiceEnrollment（PURE + SCENE）并异步处理：
1. 上传录音 → 创建 VoiceRecording
2. 同时创建 VoiceProfile（如不存在）
3. 创建两条 PENDING 状态的 VoiceEnrollment
4. 异步处理两个 enrollment
5. 返回录音信息 + enrollment IDs

### 2.3 VoiceProfile 路由适配新模型

**文件**: `api-server/src/routes/voice-profile.ts`

- 查询 `VoiceProfile` 表而非 `User`/`AnonymousUser`
- 录音只返回最新 1 条（需求 5）
- 返回 enrollments 包含状态信息供前端轮询

### 2.4 Enrollment 作废路由适配

**文件**: `api-server/src/routes/enrollment-invalidate.ts`

- 更新 `VoiceProfile` 表而非 `User`/`AnonymousUser`

### 2.5 新增 enrollment 状态轮询端点（可选）

可在 `voice-profile.ts` GET 响应中直接包含 enrollments 状态信息，无需单独端点。

## 五、执行顺序

1. **Schema 变更** → 两个 `schema.prisma` 同步修改，执行迁移
2. **后端路由改造** → VoiceProfile 查询、异步 enrollment、录音上传自动建声
3. **前端类型更新** → types.ts
4. **前端组件重构** → UserHeader、WorkspaceHeader 简化、RecordingPanel 重构
5. **前端状态管理** → use-voice-studio-state.ts 轮询逻辑
6. **文案更新** → 所有相关组件
7. **验证** → `npm run lint && npm run typecheck && npm run build`

## 六、风险与注意事项

- 数据迁移：需确保现有 User/AnonymousUser 的 active enrollment ID 正确迁移到 VoiceProfile
- 异步处理：后端使用 `void` 启动异步函数，需确保错误被正确捕获和记录
- 轮询频率：前端每 2 秒轮询一次，需设置最大轮询次数/超时（如 60 次 = 2 分钟）
- 并发安全：VoiceProfile 的创建和更新需处理并发场景（事务隔离）
- 前端构建验证：所有变更需通过 `npm run lint && npm run typecheck && npm run build`
